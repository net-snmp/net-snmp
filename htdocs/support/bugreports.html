Bug Reports

The SourceForge bug database is used to report bugs in Net-SNMP or UCD-SNMP.

Before reporting a bug, please search the SourceForge bug database and the SourceForge patch database to see if the bug has already been reported or fixed.

When posting a bug, please be sure to provide at a minimum, the following information:

  • The version of Net-SNMP / UCD-SNMP used. If you are using the CVS version, be sure to include the date the source was downloaded.
  • The operating system version (eg: output from 'uname -a').
  • If applicable, the operating system distribution (eg: SuSE Linux 9.1)
  • Sample output demonstrating the bug.
  • Detailed steps on how to reproduce the problem including exact commands used.
  • If other libraries are involved such as OpenSSL, include the library version.
  • If possible a stack trace (see below) if it had crashed

Submitting a stack trace

When your agent, snmptrapd or application crashes here are instructions for submitting a stack trace using GDB which will greatly help us debug the problem.

  • First, you either need to be able to reproduce it or have a core file as a result of the crash. You either need to call gdb appropriately depending on which you have:
    • With a core file:
      1. % gdb /path/to/ProgramThatCrashed corefile
      After this a bunch of gdb lines will go by, ending in a "gdb>" prompt.
    • Reproducing it the next time inside GDB:
      1. % gdb /path/to/ProgramThatCrashed
      2. [wait for the "gdb> prompt...]
      3. gdb> run ARGUMENTS-TO-THE-PROGRAM
      4. [Get program to crash now...]

      Use whatever program arguments you used when getting it to crash the first time. For daemons like snmpd and snmptrapd, be sure to include the -f flag to get it to not fork into the background though.

  • Once you're program is at the stopping point, the most useful thing you can do to us the results of the "where" and "list" commands:
    • gdb> where
    • [stack trace sent to your screen. Include it in the bug report]
    • gdb> list
    • [source code listing may be shown. Include it too if possible.]
